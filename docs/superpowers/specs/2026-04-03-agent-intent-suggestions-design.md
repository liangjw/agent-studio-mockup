# Agent Intent Suggestions Design

## Overview

After each AI reply in the chat, display 1-4 quick-action buttons that represent the user's likely next intent. Buttons are auto-generated by the same LLM call, not hardcoded.

## Architecture

```
User Message → Worker /api/chat → LLM → LLM returns text + ###SUGGESTIONS JSON
                                              ↓
Frontend ChatInterface ← Worker { response, suggestions[] }
```

## Data Contract

**Request:** unchanged — same as current `/api/chat`

**Response:** adds `suggestions` field

```json
{
  "response": "要创建 Dify Agent，请提供 Dify 的服务地址。",
  "suggestions": ["确认创建", "查看架构图", "换个问题"]
}
```

`suggestions` is always an array of strings. Empty `[]` if LLM didn't generate any.

## LLM Prompt

The system prompt is unchanged. Instead, a **suffix directive** is appended to every user prompt sent to the LLM:

```
After your response, output a JSON array of 1-4 short button labels (max 6 chars each) representing the user's likely next intent. If no good suggestions, output an empty array `[]`. Follow this exact format:

###SUGGESTIONS
["按钮文字1", "按钮文字2"]
```

## Worker Changes (`src/index.ts`)

After receiving LLM response text, extract suggestions:

1. Find `###SUGGESTIONS` marker in response
2. Extract everything after it (trim whitespace)
3. `JSON.parse()` the extracted string
4. Strip the `###SUGGESTIONS` block from the displayed response text
5. Return `{ response: cleanedText, suggestions: parsedArray }`
6. On parse error or missing marker → return `suggestions: []`, log error

Regex: `###SUGGESTIONS\s*([\s\S]+)$`

## Frontend Changes (`src/components/ChatInterface.tsx`)

1. Update `sendMessage` response type to include `suggestions: string[]`
2. Store `suggestions` on the message object alongside `text`
3. After each model message, render quick-action buttons below the message content if `suggestions.length > 0`
4. Clicking a button fills the input and sends it as a new user message
5. Style: same pill/button style as existing quick-reply buttons (`Dify Agent` etc.)

## Edge Cases

- LLM omits `###SUGGESTIONS` → `suggestions: []`, normal reply shown
- LLM outputs malformed JSON → `suggestions: []`, error logged but reply shown
- `suggestions` has >4 items → cap at 4 (limit in prompt)
- `suggestions` item >6 chars → truncate to 6 chars in UI
- Empty `suggestions` array → render no buttons (no change to existing UI)
